## How to read these

Each case follows the same structure:

1. **What it is**, one paragraph of plain-English description
2. **Problem**, what the client needed solved
3. **Architecture**, the actual decisions and why
4. **Stack**, concrete tools and versions
5. **Verification**, tests, metrics, what proves it shipped
6. **Notable decisions**, the 2-3 calls that mattered most

Anything redacted is marked. NDA-bound details are described at architecture level only.
